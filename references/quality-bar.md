# Quality Bar

## 核心问题

不是“这张图漂不漂亮”，而是：

> 这张图是不是用户真正要的那类资产，而且现在就能被继续使用或直接验收？

## 通用过线标准

- 用途感明确
- 资产类型正确
- 成品度与任务合同一致
- 视觉语言统一
- 语言和文字策略符合当前任务
- 没有明显廉价 AI 味

## 必须新增的验收视角

### 1. Asset Type Fidelity

问：

- 它是不是用户要的那种资产

不是：

- “相关”
- “看起来也能解释成差不多”

而是：

- 品牌宣传图就是品牌宣传图
- README hero 就是 README hero
- onboarding visual 就是 onboarding visual

### 2. Completion Readiness

问：

- 这张图现在是不是用户可验收的完成状态

高分信号：

- 用户要成品，结果就是成品
- 不需要再解释“后续补字”
- 不需要再额外说明“这是底图”

低分信号：

- 好看，但还是半成品
- 需要额外告诉用户“这版先这样，后面再补”

### 3. Language Alignment

问：

- 文案语言是否跟随当前任务

高分信号：

- 中文任务默认中文
- 项目名保留原文但其余文字不乱切语言

低分信号：

- 中文会话里默认英文 slogan
- 出现无意义英文小字、伪 UI 英文

### 4. Contract Alignment

问：

- 输出是否仍然在最初定义的交付物合同里

低分信号：

- 成品任务被做成底图
- 品牌海报被做成建筑板
- 项目宣传图被做成泛科技壁纸

## 快速红旗

出现以下任一项时，优先认为要进入 `repair` 或 `contract_realign`：

- “好看但没法直接用”
- “这不是我要的资产类型”
- “这是底图，不是成品”
- “语言不对”
- “品牌语义跑偏”
- “像 AI 图，不像设计图”

## 资产类型附加判断

### 品牌 / 招募 / 发布图

过线信号：

- 一眼能看出是发布资产
- 文案层级完整
- 可直接用于仓库宣传或社媒发布

不过线信号：

- 只有氛围，没有资产类型
- 文案不完整或语言不对
- 更像底图或概念板

### README Hero

过线信号：

- 明显服务仓库首页
- 复制区、安全区和主体关系清楚

不过线信号：

- 更像社媒海报而不是 hero

### UI / App 资产

过线信号：

- 场景和宿主产品语境一致

不过线信号：

- 更像独立海报

## 结论规则

当下面任一项明显失守时，不应称为“已经达到目标”：

- 资产类型错误
- 成品度错误
- 语言不一致
- 合同边界错误
- 好看但没法用
