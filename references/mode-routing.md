# Mode Routing

## 目标

在 `direct`、`brief-first`、`repair` 三条路径中选对一条。

## 1. direct

直接进入生成，当且仅当下面大部分都成立：

- 用户用途明确
- 主体明确
- 场景明确
- 风格方向基本明确
- 没有明显高风险文字、品牌或界面结构要求

典型例子：

- “帮我做一张极简风的咖啡机产品首图，白底、偏高端、适合电商首页”
- “做一张 SaaS 产品登录页 mockup，浅色、B2B、整洁现代”

## 2. brief-first

先问 1 到 3 个问题，当这些缺失会显著改变结果：

- 资产用途不清楚
- 用户要“像设计团队会用”，但没说清是广告图、信息流图还是 UI 概念图
- 需要真实文字、品牌元素或特定受众语气
- UI 图要求较高，但没有说明平台、页面类型或 fidelity

优先追问字段：

1. 这张图最终用在哪里
2. 更偏产品广告、社媒视觉还是 UI/App 方案图
3. 最不能跑偏的是内容、构图、风格还是商业气质

## 3. repair

满足以下任一项就优先进入修复模式：

- 用户明确说“上一版不对”
- 已经生成过，但“太 AI”“不像产品图”“不像真实 UI”
- 用户给出已有结果并要求调整

repair 模式下先做两件事：

1. 判断失败类型
2. 只改最关键的一到两个变量

不要直接把整段 prompt 推翻重写。

## 快速判断口诀

- 明确且低风险：`direct`
- 缺少少量关键 brief：`brief-first`
- 已有结果但跑偏：`repair`
