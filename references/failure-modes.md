# Failure Modes

## 为什么先沉淀失败

这个 Skill 的长期价值，不在于收集了多少“看起来不错”的 prompt，而在于知道：

- 哪类图最容易失败
- 为什么失败
- 下一轮最小该怎么改

## 1. 只有风格，没有用途

症状：

- prompt 很多形容词
- 但看不出这张图是给谁用、用在哪里

根因：

- `Asset type` 或 `Objective` 缺失

纠偏：

- 补资产用途
- 补业务目标
- 减少纯风格词堆叠

## 2. 产品图不商品化

症状：

- 画面挺漂亮
- 但不像商业商品图

根因：

- 构图、材质、留白没有服务商品展示

纠偏：

- 强化商品摄影语言
- 补主体、镜头、留白和背景的商业意图

## 3. 社媒图像壁纸，不像传播素材

症状：

- 视觉好看
- 但不像 feed creative 或 campaign asset

根因：

- 缺少平台语境、标题区、传播意图

纠偏：

- 补“平台 + 用途 + 留白策略”
- 强化信息区与视觉区关系

## 4. UI mockup 不可信

症状：

- 像 UI 风格图
- 但不像真界面

根因：

- 没写清页面类型、fidelity、层级和平台

纠偏：

- 指定 screen type
- 指定 fidelity
- 限制组件数量和布局复杂度

## 5. App 素材图喧宾夺主

症状：

- 图本身很抢眼
- 但难嵌进产品

根因：

- 没把“服务产品系统”当成约束

纠偏：

- 补产品语境
- 降低孤立艺术表达
- 强调系统兼容性

## 6. 社媒图出现未要求的额外文案

症状：

- 主标题基本正确
- 但自动冒出了 badge、副标题、栏目名或其他未要求文字

根因：

- `Text (verbatim)` 约束不够强
- prompt 只说了想要传播感，没有说“只能出现这些字”

纠偏：

- 明确写 `only show the exact headline text`
- 在 repair 中把 `Change only` 收敛为“移除额外文案，保留布局方向”
- 对社媒图补“平台适配”而不是“额外营销标签”

## 7. UI mockup 伪造品牌名或 logo

症状：

- 页面结构基本可信
- 但自动出现了并未要求的品牌名、logo mark、账户身份

根因：

- 没有显式限制品牌身份生成
- 模型会用品牌元素补齐“产品感”

纠偏：

- 补 `no brand name, no logo mark, no fake company identity`
- 如果只是做中性 mockup，强调 `generic product mockup`
- repair 时优先只移除品牌身份，不要同时重做整个布局

## 8. 追问应该发生却没发生

症状：

- 用户说“符合预期很重要”
- 但没有问关键 brief 就直接生成

根因：

- 路由判断过度乐观

纠偏：

- 回到 `brief-first`
- 只问最影响结果的 1 到 3 个问题

## 9. 本该直接生成却问太多

症状：

- 用户需求已经明确
- 但流程被不必要追问拖慢

根因：

- 路由判断过度保守

纠偏：

- 回到 `direct`
- 对可合理假设的字段直接补全

## 10. 修图时一次改太多

症状：

- 一轮修完，无法判断哪项改动真正有效

根因：

- 没有单变量迭代

纠偏：

- 一轮只改一到两个关键变量
- 保留 change-only 和 invariants

## 快速排查顺序

1. 这张图的用途是否写清
2. 当前类型是否分类正确
3. 最不能跑偏的点是否明确
4. 是生成问题还是修图问题
5. 这次该改一个变量还是多个变量
