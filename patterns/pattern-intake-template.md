# Pattern Intake Template

## Purpose

不是每次成功都值得升级成 pattern。

先回答这些问题：

- `scene`
  - 这类任务常发生在什么场景？
- `trigger`
  - 哪些信号说明该用这个打法？
- `failure_before`
  - 原来常见的失败是什么？
- `intervention`
  - 核心纠偏动作是什么？
- `transfer_rule`
  - 为什么它能迁移到其他任务？

## Promotion Rule

只有下面三项都明确，才建议升级：

1. 触发信号清楚
2. 纠偏动作稳定
3. 迁移规律清楚
